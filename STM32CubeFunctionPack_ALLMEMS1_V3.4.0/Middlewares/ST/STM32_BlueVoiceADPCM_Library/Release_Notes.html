Release Notes for BlueVoiceADPCM

Voice over Bluetooth Low Enegy Library for STM32 and BlueNRG-MS

Copyright 2017 STMicroelectronics



What is BlueVoiceADPCM library?

  • BlueVoiceADPCM library implements a 'vendor specific profile' enabling voice communication over Bluetooth Low Energy.
  • BlueVoice profile defines a BLE service including one characteristic for audio transmission and one for synchronization. In a half-duplex system, both sides of the communication (Central and Peripheral) can act as servers of information. Periodical notifications containing compressed audio data are sent from one Server to one Client depending on the selected channel: Central to Peripheral or Peripheral to Central. BlueVoiceADPCM middleware is responsible for audio encoding and periodical data transmission on Server side and for decoding of received voice data on Client side. 

BlueVoiceADPCM features

  • Half-duplex or simplex voice over Bluetooth Low Energy communication profile.
  • Based on ST very low power Bluetooth Low Energy single-mode Network Processor, compliant with Bluetooth specifications core 4.1 (BlueNRG-MS).
  • Designed for optimal performance in applications using ST’s digital MEMS microphones.
  • Digital audio signal processing.
  • Easy portability across different STM32 MCU families thanks to modular architecture and STM32Cube.
  • Free user-friendly license terms.
This Library includes one header file bluevoice_adpcm.h and binary/object codes running on STM32 devices based on CortexM0 and CortexM4F cores. The naming format is bluevoiceADPCM_200_CMx_C_ot.a/.lib, where:
  • x: CortexMx core class (CM0, CM4 and CM4F)
  • C: compiler (IAR, Keil and GCC)

 

Update History

V2.0.0 / 18-May-2017

Main Changes

  • First version of the library published under the ODE initiative. Previously released as part of OSX.
  • Redesigned library architecture. framework.
  • Added configuration parameters.
  • Changed sending data mechanism.  
  • Different working mode: Transmitter, Receiver, Half-Duplex.
  • Compatible with ARM Cortex M4 and M0+
